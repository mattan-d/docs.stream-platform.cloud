מה עושה תוסף Stream?

תוסף ה‑Stream למודל הוא תוסף מקומי (local plugin) שמחבר בין מערכות הוידאו החיצוניות (Zoom, Microsoft Teams, Unicko) לבין קורסי ה‑Moodle, ומטפל אוטומטית בהקלטות המפגשים.

המרצה יוצר מפגש מקוון (למשל Zoom) מתוך הקורס, מקליט לענן, ותוסף Stream מאתר את ההקלטה, יוצר עבורה משאב במודל ומשייך אותה לקורס ולחלק הנכון – ללא הורדה/העלאה ידנית.

1. יכולות עיקריות

  • איתור אוטומטי של הקלטות אונליין – חיבור ל‑Zoom / Teams / Unicko ובדיקה מחזורית אם קיימות הקלטות חדשות שהסתיימו ועברו עיבוד.
  • הטמעה אוטומטית בקורסים במודל – יצירה או שימוש חוזר בפעילות Moodle (למשל Page / LTI / Zoom / msteams) להצגת ההקלטה, ומיקום המשאב בקורס ובחלק (section) הנכונים.
  • מניעת כפילויות – זיהוי הקלטות כפולות (למשל לפי Zoom UUID) כדי שלא יוטמעו מספר פעמים.
  • התראות לסטודנטים (אופציונלי) – שליחת הודעה למשתתפי הקורס כשמוטמעת הקלטה חדשה, בעזרת scheduled tasks ו‑adhoc tasks של Moodle.

2. למנהלי מערכת (Site Administrators)

2.1 התקנה

  1. העתיקו את תיקיית התוסף אל local/stream בתוך קוד המודל.
  2. היכנסו ל‑Site administration → Notifications ובצעו את שלבי השדרוג הרגילים.
  3. לאחר ההתקנה, עברו להגדרות התוסף.

2.2 הגדרות אינטגרציה

נתיב: Site administration → Plugins → Local plugins → Stream.

  • Platform – בחירת פלטפורמת ההקלטות: Zoom / Microsoft Teams / Unicko (LTI).
  • API / Integration settings – הזנת פרטי גישה:
    • ב‑Zoom: מפתחות/סודות (JWT / OAuth / Server‑to‑server לפי ההתקנה), והרשאה ל‑cloud recordings.
    • ב‑Teams / Unicko: כתובות, מזהים ופרטי LTI/אינטגרציה מתאימים.
  • Based grouping / duplicate handling – קיבוץ לפי מזהה פגישה ייחודי (Zoom UUID וכו') כדי להימנע מהטמעת אותה הקלטה מספר פעמים.
  • Embed order – האם למקם את משאב ההקלטה ממש מתחת לפעילות המקורית (למשל פעילות Zoom) או רק באותו section.

2.3 משימת cron (Scheduled task)

  • ודאו שהמשימה local_stream\task\embed מופעלת:
    • Site administration → Server → Scheduled tasks.
    • חפשו Embedded recordings (local_stream).
    • ודאו שהיא פעילה ומוגדרת לריצה בתדירות מתאימה (למשל כל 5–15 דקות או פעם בשעה).

2.4 זרימת Zoom – מבט על

  1. מרצה יוצר פעילות Zoom בקורס במודל.
  2. הפגישה מתקיימת ומוקלטת ל‑ענן של Zoom.
  3. משימת Stream רצה, מושכת רשימת הקלטות מ‑Zoom ומתאימה אותן לקורס ולפעילות Zoom הרלוונטיים.
  4. לכל הקלטה חדשה נוצרת פעילות (לרוב Page) עם נגן/קישור להקלטה, המוצבת באותו קורס ו‑section, ואופציונלית ישירות מתחת לפעילות Zoom.
  5. אם התראות מופעלות – משתתפי הקורס מקבלים הודעה על הקלטה חדשה.

3. למרצים / מנהלי קורס

3.1 מה נדרש מכם?

  1. יצירת פעילות Zoom בקורס – הוספת פעילות Zoom חדשה לקורס והגדרת הזמן, הנושא וההגדרות הרגילות.
  2. הקלטת המפגש לענן – ודאו שההקלטה מוגדרת כ‑cloud recording.
  3. המתנה להטמעה אוטומטית – לאחר סיום עיבוד ההקלטה ב‑Zoom, משימת Stream תאתר אותה ותיצור פעילות הקלטה אוטומטית בקורס.
  4. בדיקת דף הקורס – ההקלטה תופיע באותו section, בדרך כלל מתחת לפעילות ה‑Zoom.

3.2 הערות חשובות

  • אין צורך להוריד ולהעלות ידנית הקלטות מ‑Zoom למודל.
  • אם הקלטה לא מופיעה:
    • בדקו שההקלטה באמת נשמרה ל‑cloud.
    • פנו למנהל האתר לוודא שהתוסף מותקן ומוגדר, ושהמשימה local_stream\task\embed רצה ללא שגיאות.

4. לסטודנטים

  • לאחר שהמרצה מקליט מפגש מקוון וההקלטה מוכנה, היא תופיע אוטומטית בקורס הרלוונטי.
  • ברוב המקרים תמצאו אותה באותו חלק בקורס שבו נמצא קישור ה‑Zoom/Teams, לעיתים ממש מתחתיו.
  • יתכן ותקבלו התראה במודל כשמוספת הקלטה חדשה (בהתאם להגדרות האתר).
  • לחיצה על פעילות ההקלטה תפתח את הנגן ותאפשר צפייה בשיעור.

5. פתרון תקלות (תקציר)

5.1 ההקלטה לא הוטמעה בקורס

  • בדקו ש‑Embedded recordings (local_stream) פעילה ורצה בלוח המשימות של Moodle.
  • ב‑Zoom – ודאו ש‑cloud recordings מופעלות, וה‑API credentials נכונים.

5.2 ההקלטה הופיעה בקורס/חלק שגוי

  • ב‑Teams / Unicko – בדקו את הגדרות המיפוי בין מפגשים לקורסים/sections.
  • בדקו בהגדרות התוסף כיצד נקבעים ה‑section והסדר (Embed order).

5.3 כפילויות של אותה הקלטה

  • וודאו שהגדרת ה‑“based grouping” מכוונת נכון, כך שגרסאות שונות של אותה הקלטה לא יוטמעו שוב ושוב.

אם הבעיות נמשכות, מומלץ להפעיל Debugging במודל, להריץ ידנית את המשימה local_stream\task\embed ולבדוק בלוגים האם מופיעות הודעות שגיאה.

מסמך זה מתאר את השימוש הבסיסי ב‑Stream. ייתכן שהמערכת שלכם כוללת התאמות ספציפיות לארגון – במקרה של הבדלים, הממשק בפועל הוא הקובע.

תמיכה: לשאלות טכניות ותמיכה במערכת Stream ניתן לפנות לחברת מ.ר סנטריקאפ בע״מ בדוא״ל support@centricapp.co.il.
